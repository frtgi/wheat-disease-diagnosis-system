## 🚀 快速上手（3 分钟）

### 第 1 步：打开系统
在浏览器中访问系统地址（由您的管理员提供），例如：
```
http://your-server-ip:端口
```
使用您的账号登录（如果没有账号，请联系管理员注册）。

### 第 2 步：拍摄/上传照片
1. 点击首页的 **「开始诊断」** 按钮
2. 选择 **「上传图片」** 或 **「拍照」**（如果使用手机）
3. 选择一张清晰的小麦叶片/穗部照片
4. 点击 **「确认上传」**

> 💡 **拍照小技巧**:
> - 在**自然光**下拍摄，避免强烈阳光直射
> - 手机距离植株 **20-30 厘米**，让病害部位填满画面
> - 确保 **叶片平整**，不要有遮挡
> - 如果可能，分别拍摄 **正面** 和 **背面**

### 第 3 步：等待诊断结果
系统会自动进行分析，您将看到实时进度：
1. 🔍 **正在检测图像** — 系统识别照片中的小麦部位
2. 🧠 **AI 分析中** — 人工智能正在分析病害特征
3. 📊 **生成报告** — 汇总诊断结果和防治建议

整个过程通常在 **几秒到半分钟** 内完成。

## 场景一：拍张照片识病害

### 适用情况
您在田间发现了异常的小麦叶片，拍了照片，想快速知道是什么病害以及如何防治。

### 操作步骤

#### 步骤 1：进入图像诊断
1. 登录系统后，点击左侧菜单的 **「智能诊断」**
2. 选择 **「图像诊断」** 选项卡

#### 步骤 2：上传照片
```
┌─────────────────────────────────┐
│  📷 点击此处上传图片             │
│                                 │
│  支持 JPG / PNG / WEBP 格式     │
│  图片大小不超过 10 MB            │
│                                 │
│  或拖拽图片到此处               │
└─────────────────────────────────┘
```

#### 步骤 3：（可选）补充信息
如果您愿意，可以填写以下信息帮助 AI 更准确地判断：
- **症状描述**：例如 "叶片上有黄色条纹"、"穗部发黑"
- **生长阶段**：例如 "拔节期"、"抽穗期"、"灌浆期"
- **种植地点**：例如 "河南省周口市"

> ⚠️ 这些信息是**可选的**，不填写也能进行诊断。但填写后通常能得到更准确的结果。

#### 步骤 4：查看诊断结果

诊断完成后，您将看到类似以下的结果：

```
┌──────────────────────────────────────────┐
│  📋 诊断结果                              │
│                                          │
│  🔴 最可能的病害：**小麦条锈病**           │
│  置信度：92%（非常确定）                  │
│  严重程度：高                             │
│                                          │
│  📊 其他可能性：                          │
│  • 叶锈病 — 65% 的可能性                 │
│  • 白粉病 — 23% 的可能性                 │
│                                          │
│  💡 防治建议：                            │
│  1. 选用三唑类杀菌剂（如烯唑醇、戊唑醇）  │
│  2. 建议在发病初期喷施，间隔 7-10 天      │
│  3. 注意轮换用药，避免产生抗药性          │
│                                          │
│  📚 相关知识：[点击查看详细资料]          │
└──────────────────────────────────────────┘
```

#### 结果解读
| 字段 | 含义 | 如何使用 |
|------|------|----------|
| 最可能的病害 | AI 判断最可能是哪种病 | 作为主要参考 |
| 置信度 | AI 对判断的把握程度 | >80% 通常很可靠 |
| 严重程度 | 病害的轻重程度 | 决定是否需要紧急处理 |
| 其他可能性 | 其他可能的病害 | 用于对比排除 |
| 防治建议 | 专家推荐的防治方法 | 直接指导用药 |
| 相关知识 | 来自农业知识库的资料 | 深入了解该病害 |

## 场景二：图文结合更精准

### 适用情况
您不仅拍了照片，还想补充描述田间观察到的具体情况（比如天气、发病时间等），以获得更精准的诊断结果。

### 操作步骤

#### 步骤 1：进入多模态诊断
1. 点击左侧菜单的 **「智能诊断」**
2. 选择 **「多模态诊断」** 选项卡（同时支持图片+文字）

#### 步骤 2：上传照片并填写信息

**上传照片部分**：
- 与场景一相同，上传清晰的照片

**文字描述部分**（建议包含以下信息）：

**📝 症状描述模板**：
```
【必填】发病部位：叶片 / 叶鞘 / 茎秆 / 穗部 / 籽粒
【必填】主要症状：颜色 + 形状 + 大小
【选填】发病时间：什么时候发现的？
【选填】发病面积：大概多少亩？占多少比例？
【选填】近期天气：温度多少？有没有下雨？
【选填】之前用过什么药？效果如何？
```

**示例输入**：
```
发病部位：叶片
主要症状：叶片上有黄色条纹状斑点，沿叶脉排列，
         用手触摸会有黄色粉末粘在手上
发病时间：3天前发现，现在越来越多了
发病面积：大约5亩地，占了30%
近期天气：最近一直阴雨，温度在10-15度左右
之前用药：还没有用药
```

#### 步骤 3：提交诊断
点击 **「开始诊断」** 按钮，系统会综合分析图像和文字信息。

#### 步骤 4：查看增强版诊断结果

与纯图像诊断相比，多模态诊断会提供更详细的信息：


| 对比项 | 仅图像诊断 | 图像+文字诊断 |

## 场景三：查看历史记录

### 适用情况
- 您想回顾之前的诊断记录
- 您想对比不同时间的病情变化
- 您需要导出诊断报告用于存档或汇报
- 您想统计某个时间段内的病害发生情况

### 操作步骤

#### 步骤 1：进入历史记录
1. 点击左侧菜单的 **「历史记录」**
2. 系统会显示您所有的诊断历史列表

#### 步骤 2：筛选和查找记录

系统支持多种筛选方式：

**按时间筛选**：
- 今天 / 最近7天 / 最近30天 / 自定义日期范围

**按病害类型筛选**：
- 全部病害 / 条锈病 / 叶锈病 / 白粉病 / 赤霉病 ...

**按诊断方式筛选**：
- 全部 / 图像诊断 / 多模态诊断

**搜索功能**：
- 可以搜索关键词（如"条锈"、"黄斑"）

#### 步骤 3：查看单条记录详情

点击任意一条记录，您可以查看：

**基本信息**：
- 诊断时间
- 诊断方式（图像/多模态）
- 上传的照片（可放大查看）

**诊断结果**：
- 判断的病害名称
- 置信度和严重程度
- 完整的防治建议

**原始数据**：
- 如果是多模态诊断，可以看到当时填写的症状描述
- 环境信息（温度、湿度、生长阶段等）

**操作按钮**：
- 📋 **复制报告** — 复制完整的诊断报告文本
- 📤 **导出为PDF** — 生成PDF文件保存或打印
- 🔄 **重新诊断** — 用同样的照片重新分析（模型更新后有用）
- ❓ **反馈结果** — 标记这次诊断是否准确（帮助改进系统）

#### 步骤 4：批量导出记录

**导出为 Excel 表格**：
1. 勾选要导出的记录（或全选）
2. 点击 **「导出」** 按钮
3. 选择 **「Excel 格式」**
4. 系统会生成 `.xlsx` 文件供下载

Excel 表格包含以下列：
| 列名 | 说明 |
|------|------|
| 诊断ID | 唯一编号 |
| 诊断时间 | 年月日时分秒 |
| 诊断方式 | 图像/多模态 |
| 病害名称 | 判断结果 |
| 置信度 | 百分比 |
| 严重程度 | 轻/中/重 |
| 防治建议摘要 | 主要建议 |
| 用户反馈 | 是否正确 |

**导出为 PDF 报告**：
1. 选择单条或多条记录
2. 点击 **「导出 PDF 报告」**
3. 每条记录会生成独立的 PDF 文件
4. PDF 包含照片、诊断结果、防治建议等完整信息

#### 步骤 5：统计分析（高级功能）

在历史记录页面顶部，您可以看到统计图表：

**📊 诊断量统计**：
- 柱状图显示每天/每周的诊断数量趋势

**🥧 病害分布饼图**：
- 显示各种病害的诊断占比
- 帮助了解本地常见病害类型

**📈 准确率趋势**：
- 如果您提供了反馈，可以看到系统的诊断准确率变化

> 💡 **实用技巧**：
> - 定期导出历史记录，建立自己的**病害档案**
> - 对比不同年份同一时期的记录，可以**预测病害发生规律**
> - 将诊断报告打印出来，方便**田间作业时参考**

## 场景四：了解病害详情

### 适用情况
- 您想深入了解某种小麦病害的特征和防治方法
- 您想学习如何区分相似的病害
- 您需要查找特定药剂的用法用量
- 您想了解病害的发生规律和预防措施

### 操作步骤

#### 步骤 1：进入知识库
1. 点击左侧菜单的 **「知识库」** 或 **「病害百科」**
2. 进入农业专业知识查询界面

#### 步骤 2：浏览病害列表

系统支持 **17 种常见小麦病害/虫害** 的详细资料：

**真菌性病害（13种）**：
- 条锈病、叶锈病、茎锈病
- 白粉病、赤霉病
- 根腐病、纹枯病
- 叶斑病、壳针孢叶斑病
- 斑点叶斑病、褐斑病
- 黑粉病、小麦爆发病

**昆虫害虫（3种）**：
- 蚜虫、螨虫、茎蝇

**健康状态**：
- 健康（无病害）

#### 步骤 3：查看某种病害的详细信息

点击任意一种病害名称，您将看到：

**📸 典型症状图谱**：
- 多张真实病害照片
- 标注关键特征部位
- 不同发病阶段对比图

**📋 基本信息**：
```
病害名称：小麦条锈病
病原菌：条形柄锈菌 (Puccinia striiformis)
病害类型：真菌性病害
危害部位：主要危害叶片，也可危害叶鞘、茎秆和穗部
```

**🔬 症状识别要点**：
- **早期症状**：叶片上出现褪绿斑点
- **中期症状**：黄色条状孢子堆，沿叶脉排列
- **后期症状**：孢子堆连成片，叶片枯死
- **典型特征**：用手触摸会有黄色粉末（夏孢子）

**🌡️ 发生规律**：
- **适宜温度**：9-16°C（最适 12-14°C）
- **适宜湿度**：高湿度（相对湿度 >80%）
- **易发时期**：春季（3-5月）、秋季（9-11月）
- **传播途径**：气流传播、远距离传播能力强

**💊 防治方案**：

**化学防治**：
| 药剂名称 | 剂型 | 使用浓度 | 施药方法 | 安全间隔期 |
|---------|------|---------|---------|-----------|
| 三唑酮 | 15%可湿性粉剂 | 1000倍液 | 喷雾 | 21天 |
| 戊唑醇 | 10%可湿性粉剂 | 1500倍液 | 喷雾 | 28天 |
| 烯唑醇 | 12.5%乳油 | 2000倍液 | 喷雾 | 28天 |
| 苯醚甲环唑 | 10%水分散粒剂 | 1500倍液 | 喷雾 | 21天 |

**生物防治**：
- 枯草芽孢杆菌制剂
- 木霉菌制剂
- 哈茨木霉

**农业防治**：
1. 选用抗病品种（如：济麦22、周麦27等）
2. 适期晚播（避免早播遇低温高湿）
3. 合理密植，改善通风透光
4. 控制氮肥用量，增施磷钾肥
5. 清除田间病残体，减少菌源
6. 药剂拌种（用三唑类杀菌剂拌种）

**预防措施**：
- 秋播前进行种子处理
- 加强田间巡查，早期发现
- 注意天气预报，提前预防
- 轮作倒茬（与非禾本科作物轮作）

**⚠️ 注意事项**：
- 不要长期单一使用同一种药剂（避免抗药性）
- 严格按照推荐浓度使用（不要随意加大剂量）
- 注意施药安全（戴口罩、手套，避免在高温时段施药）
- 轮换使用不同作用机制的药剂

#### 步骤 4：使用搜索功能

如果不确定病害名称，可以使用搜索：

**关键词搜索**：
- 输入症状："黄斑"、"白粉"、"黑穗"
- 输入部位："叶片"、"穗部"、"茎秆"
- 输入季节："春季病害"、"夏季"

**智能问答**：
在搜索框中输入自然语言问题：
```
示例问题：
• "叶片上有黄色条纹是什么病？"
• "条锈病和白粉病怎么区别？"
• "赤霉病用什么药最好？"
• "小麦什么时候最容易生病？"
• "如何预防小麦病害？"
```

系统会基于知识库给出详细的回答。

#### 步骤 5：病害对比功能

对于容易混淆的病害，系统提供 **对比视图**：

**常见易混淆病害组**：
1. **三种锈病对比**：条锈病 vs 叶锈病 vs 茎锈病
2. **叶部病害对比**：白粉病 vs 叶斑病 vs 褐斑病
3. **穗部病害对比**：赤霉病 vs 黑粉病

对比表格示例（条锈病 vs 叶锈病）：

| 特征 | 条锈病 | 叶锈病 |
|------|--------|--------|
| **病斑形状** | 条状（沿叶脉） | 圆形或椭圆形（散生）|
| **病斑颜色** | 黄色或橙黄色 | 红褐色或橙色 |
| **排列方式** | 规则排列成行 | 不规则散生分布 |
| **发生部位** | 叶片正面为主 | 叶片正反面都有 |
| **适宜温度** | 较低（9-16°C）| 较高（15-25°C）|
| **发生时期** | 早春为主 | 春末夏初为主 |
| **触摸手感** | 有粉末感 | 有粉末感 |

> 💡 **学习建议**：

## ❓ 常见问题 FAQ

### 📸 关于拍照和上传

**Q1: 应该拍什么样的照片？**

A: 为了获得准确的诊断结果，请注意以下几点：
- ✅ **拍摄对象**：拍摄有明显症状的小麦叶片、茎秆或穗部
- ✅ **光线条件**：选择自然光充足的时候（避免正午强光直射）
- ✅ **拍摄距离**：手机距离植株 20-30 厘米，让病变部位清晰可见
- ✅ **画面构图**：病变区域应占据画面 60% 以上
- ✅ **对焦清晰**：确保照片清晰不模糊
- ❌ **避免**：光线过暗、画面模糊、背景杂乱、叶片重叠遮挡

**Q2: 支持哪些图片格式？**

A: 系统支持以下格式：
- **JPG/JPEG**（推荐）：兼容性好，文件较小
- **PNG**：支持透明背景，质量较高
- **WEBP**：新一代格式，体积小质量好

**Q3: 图片大小有限制吗？**

A: 有的。为了确保诊断速度和质量：
- 单张图片不超过 **10 MB**
- 推荐分辨率：**800×600 像素以上**
- 手机直接拍摄的原图通常都可以满足要求

**Q4: 可以一次上传多张图片吗？**

A: 可以！系统支持两种模式：
- **单张诊断**：一次分析一张图片，适合重点病例
- **批量诊断**：最多可同时上传 50 张图片，适合大面积普查

**Q5: 手机能使用吗？**

A: 当然可以！系统完全支持手机浏览器：
- 📱 支持手机拍照直接上传
- 📱 界面自动适配手机屏幕大小
- 📱 支持主流浏览器（微信内置浏览器、Safari、Chrome等）
- 📱 建议：使用 WiFi 或 4G/5G 网络，避免流量超支

### 🔍 关于诊断结果

**Q6: 诊断结果准确吗？我能相信吗？**

A: 系统经过大量样本训练和验证：
- 对于常见的 **17 种小麦病害**，Top-1 准确率目标 > **85%**
- 置信度 > **90%** 的结果通常非常可靠
- 系统会同时列出其他可能性，帮您全面了解

> ⚠️ **重要提示**：
> - AI 诊断是**辅助工具**，不能完全替代专业农技人员的判断
> - 建议将 AI 结果作为**初步参考**，最终决策请结合实际情况
> - 如有疑问，请联系当地农技站进行实地鉴定
> - 对于重大病虫害决策，务必咨询专业人士

**Q7: 什么是置信度？怎么看？**

A: **置信度**表示 AI 对判断结果的把握程度：
- **90-100%**：非常确定，结果高度可信
- **70-89%**：比较确定，结果可信，但建议结合其他可能性
- **50-69%**：有一定把握，建议仔细对比其他可能性
- **<50%**：不确定性较大，建议重新拍照或咨询专家

**Q8: 诊断结果显示了多种可能，我该听谁的？**

A: 这是正常的！系统会按照可能性从高到低排序：
1. **首选结果**：概率最高的病害，通常是正确的
2. **备选结果**：其他可能的病害，用于排除
3. **建议做法**：
   - 如果首选置信度 >85%，一般可以直接参考
   - 如果多个结果置信度接近，建议：
     - 重新拍摄更清晰的照片
     - 补充更多症状描述（使用多模态诊断）
     - 对比知识库中的症状图，自行判断

**Q9: 诊断需要多长时间？**

A: 通常很快：
- **简单病例**：3-10 秒
- **复杂病例**：10-30 秒
- **批量诊断**：取决于图片数量，平均每张 5-15 秒

影响速度的因素：
- 图片大小和清晰度
- 服务器当前负载（高峰期可能稍慢）
- 网络状况

> 💡 **提示**：诊断过程中您会看到实时进度提示，请耐心等待，不要重复点击。

**Q10: 为什么有时候诊断失败？**

A: 可能的原因及解决方法：

| 可能原因 | 解决方法 |
|---------|---------|
| 图片模糊或不清晰 | 重新拍摄清晰的照片 |
| 光线太暗或太亮 | 在自然光下重新拍摄 |
| 拍摄的不是小麦 | 确保拍摄的是小麦植株 |
| 病症不明显或太轻微 | 等病症明显后再拍，或拍摄多个角度 |
| 图片格式不支持 | 使用 JPG/PNG/WEBP 格式 |
| 网络连接问题 | 检查网络，稍后重试 |
| 服务器繁忙 | 稍等几分钟再试 |

### 💊 关于防治建议

**Q11: 系统推荐的农药安全吗？**

A: 系统推荐的农药都是：
- 经过国家登记的**正规农药**
- 来自权威农业部门发布的**防治指南**
- 符合**绿色防控**理念，优先推荐低毒高效药剂

> ⚠️ **用药安全提醒**：
> - 严格按照推荐**浓度**使用，不要随意加大剂量
> - 注意**安全间隔期**（最后一次施药到收获的天数）
> - 施药时做好个人防护（口罩、手套、长袖衣裤）
> - 避免**大风天**或**高温时段**（中午）施药
> - 剩余药液要**妥善处理**，不要倒入水源
> - 购买农药时选择**正规渠道**，注意查看生产日期和保质期

**Q12: 防治建议中的"化学防治"、"生物防治"、"农业防治"有什么区别？**

A: 三种防治方式各有特点：

| 类型 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| **化学防治** | 使用杀虫剂/杀菌剂 | 见效快、效果好 | 有残留、易产生抗药性 | 病害爆发期紧急控制 |
| **生物防治** | 利用有益微生物/天敌 | 环境友好、无残留 | 见效较慢、成本较高 | 预防期、轻度发生 |
| **农业防治** | 通过栽培管理措施 | 无污染、成本低 | 见效慢、需长期坚持 | 全程预防、综合治理 |

**推荐策略**：**预防为主，综合防治**
1. 平时以农业防治为基础（选用抗病品种、合理栽培）
2. 发病初期优先使用生物防治
3. 病害严重时合理使用化学防治
4. 注意轮换用药，延缓抗药性产生

**Q13: 系统推荐的药剂在当地买得到吗？**

A: 系统推荐的药剂都是**常用农药**，一般在当地农资店都能买到：
- 三唑类杀菌剂（三唑酮、戊唑醇、烯唑醇等）— **非常普遍**
- 甲氧基丙烯酸酯类（嘧菌酯、吡唑醚菌酯等）— **广泛使用**
- 生物制剂（枯草芽孢杆菌等）— **大型农资店有售**

> 💡 **购买建议**：
> - 到正规的**农资公司**或**供销社**购买
> - 查看**农药登记证号**和生产批号
> - 保留**购货凭证**
> - 咨询店主具体的使用方法和注意事项

## 📖 术语对照表

本对照表帮助您理解系统中出现的一些专业术语。

| 您看到的术语 | 通俗解释 | 举个例子 |
|-------------|---------|---------|
| **多模态融合** | 同时分析图片和文字信息，就像医生既看片子又听您描述症状 | 您上传照片 + 填写症状描述 = 多模态诊断 |
| **置信度** | AI 对判断结果的把握程度，范围 0-100% | 置信度 95% 就像说"我有 95% 的把握" |
| **AI / 人工智能** | 模拟人脑思维的计算机程序，能学习和判断 | 就像一个经验丰富的"数字老农" |
| **ROI (感兴趣区域)** | 照片中标记出的病害位置，会用框框标出来 | 就像在照片上圈出"这里有问题" |
| **边界框 (BBox)** | 围绕病灶画的一个矩形框，标注位置和大小 | 类似于给病灶画个方框 |
| **严重程度** | 病害的轻重级别：轻度 / 中度 / 重度 | 就像感冒分"有点感冒"和"重感冒" |
| **知识图谱** | 存储农业专业知识的智能数据库，能把相关知识关联起来 | 就像一本会思考的农业百科全书 |
| **SSE (Server-Sent Events)** | 一种让浏览器实时接收结果的技术，不用刷新页面就能看到进度 | 就像看直播，实时看到诊断进展 |
| **INT4 量化** | 让 AI 模型变小变快的技术，不影响准确性 | 就像把大箱子压缩打包，打开还是一样的东西 |
| **懒加载** | 只在需要时才加载 AI 模型，加快启动速度 | 就像按需取书，用到哪本拿哪本，不用一次性搬全部 |
| **GPU 加速** | 使用显卡（图形处理器）来加速计算，比 CPU 快很多 | 就像用跑车代替自行车跑高速 |
| **推理 (Inference)** | AI 模型进行分析判断的过程 | 就是"思考"和"得出结论"的过程 |
| **训练 (Training)** | 让 AI 学习大量样本的过程 | 就像让学生做大量练习题来学习 |
| **准确率 (Accuracy)** | AI 判断正确的比例 | 准确率 90% 表示 100 次判断中有 90 次是对的 |
| **Top-1 / Top-3** | AI 给出的最可能的前 1 个或前 3 个答案 | Top-1 是最可能的那个，Top-3 是最可能的三个 |
| **召回率 (Recall)** | 能找出所有真正病例的能力 | 就像医生能不能把所有病人都找出来 |
| **精确率 (Precision)** | 判断为病的里面有多少真的是病 | 就像医生说"你有病"，说对的概率是多少 |
| **F1 分值** | 同时考虑准确率和召回率的综合指标 | 就像考试成绩的总分 |
| **过拟合 (Overfitting)** | AI 太死记硬背，遇到新情况就不会了 | 就像学生只会做见过的题，新题型就不会 |
| **泛化能力 (Generalization)** | AI 遇到没见过的情况也能正确判断的能力 | 就像举一反三的能力 |
| **数据增强 (Augmentation)** | 通过旋转、翻转等方式增加训练数据 | 就像从一张照片变出很多张不同的照片 |
| **迁移学习 (Transfer Learning)** | 把在一个任务上学到的知识用到另一个任务上 | 就像学会了骑自行车，学摩托车就快了 |
| **微调 (Fine-tuning)** | 在已有模型基础上针对特定任务继续训练 | 就像在通用的基础上专门学习某方面的知识 |
| **LoRA** | 一种高效的微调方法，只训练少量参数 | 就像只需要复习重点，不用从头学 |
| **API (接口)** | 程序之间交流的方式，就像两个系统的"电话线" | 其他软件可以通过 API 调用我们的诊断功能 |
| **端到端 (End-to-End)** | 从输入到输出一步到位，不需要中间人工干预 | 就像全自动流水线，放进去原材料，出来就是成品 |
| **云计算 (Cloud Computing)** | 通过互联网使用远程服务器的计算能力 | 就像租用超级计算机，不用自己买 |
| **边缘计算 (Edge Computing)** | 在本地设备上进行计算，不用传到远处 | 就像在自己家里加工，不用送到远方工厂 |
| **延迟 (Latency)** | 从发出请求到收到结果的时间 | 就像打电话的"接通时间" |
| **吞吐量 (Throughput)** | 系统能处理的请求数量 | 就像工厂一天能生产多少产品 |
| **并发 (Concurrency)** | 同时处理多个请求的能力 | 就像服务员同时招待好几桌客人 |
| **容器化 (Docker)** | 把软件打包成标准化的单元，在任何环境都能运行 | 就像集装箱，不管什么车都能拉 |
| **Neo4j** | 一种专门存储关系数据的数据库系统 | 就像专门用来存放人际关系网的系统 |
| **Cypher** | Neo4j 数据库使用的查询语言 | 就像数据库的"方言" |
| **节点 (Node) / 关系 (Relation)** | 知识图谱中的基本元素 | 节点是"事物"，关系是事物之间的"联系" |
| **图谱遍历 (Graph Traversal)** | 在知识图谱中沿着关系查找相关信息 | 就像通过朋友认识朋友的朋友 |
| **语义理解 (Semantic Understanding)** | 理解文字的真实含义，而不只是关键词匹配 | 就像理解"我饿了"是想吃饭，而不是讨论"饿"这个字 |
| **BERT** | 一种先进的文字理解 AI 模型 | 就像一个精通语言的"语言学家" |
| **YOLO** | 一种快速的物体检测 AI 模型 | 就像一个一眼就能看出图里有什么的"火眼金睛" |
| **Transformer** | 现代 AI 的核心架构技术 | 就像 AI 时代的"发动机" |
| **注意力机制 (Attention)** | AI 能够重点关注重要信息的能力 | 就像人在嘈杂环境中能专注于对话 |
| **特征提取 (Feature Extraction)** | 从原始数据中找出有用的信息 | 就像从一堆沙子里淘出金子 |
| **归一化 (Normalization)** | 把数据统一到相同的范围 | 就像把不同单位的长度都转换成米 |
| **损失函数 (Loss Function)** | 衡量 AI 预测和实际差距的方法 | 就像考试后的"评分标准" |
| **优化器 (Optimizer)** | 调整 AI 参数使其更准确的方法 | 就像根据考试错题调整学习方法 |
| **学习率 (Learning Rate)** | AI 每次调整参数的幅度 | 就像每次迈步的大小，太大容易走过头，太小走得慢 |
| **批次 (Batch)** | 一次处理的数据量 | 就像一次改一批作业 |
| **Epoch (轮次)** | 把所有数据完整学习一遍叫一轮 | 就像把课本完整读一遍 |
| **Dropout** | 训练时随机忽略一些神经元，防止死记硬背 | 就像考试时遮住一些内容，考验真正的理解 |
| **正则化 (Regularization)** | 防止 AI 过于复杂的方法 | 就像奥卡姆剃刀——"如无必要，勿增实体" |
| **超参数 (Hyperparameter)** | 需要手动设置的参数（不是 AI 自己学的）| 就像机器的"旋钮"，需要人来调节 |
| **验证集 (Validation Set)** | 用来调参和选择模型的数据集 | 就像模拟考试，用来检验学习效果 |
| **测试集 (Test Set)** | 最后用来评估模型的数据集 | 就像正式考试，用来打最终分数 |
| **交叉验证 (Cross-validation)** | 把数据分成几份轮流测试，更客观地评估 | 就像多次模拟考试取平均值 |

### B. 推荐拍照设备设置

**智能手机设置建议**：
- **分辨率**：最高分辨率（通常 1200 万像素以上即可）
- **对焦模式**：开启自动对焦（AF）
- **网格线**：开启辅助构图
- **HDR**：逆光时可开启
- **定时器**：防止手抖（可选，2-3 秒）
- **关闭美颜**：确保照片真实

**相机/平板设置建议**：
- **模式**：自动模式或微距模式
- **ISO**：尽量低（100-400），保证画质
- **光圈**：适当缩小（f/4-f/8），确保清晰
- **关闭滤镜**：保持色彩真实

### D. 小麦生长阶段对照表

| 阶段 | 时间（黄淮海地区）| 特征 | 易发病害 |
|------|------------------|------|---------|
| **播种期** | 10月上中旬 | 种子萌发 | 土传病害（根腐病）|
| **苗期** | 10月下旬-11月 | 2-4 片叶 | 蚜虫、红蜘蛛 |
| **越冬期** | 12月-次年2月 | 生长缓慢 | 冻害、雪腐病 |
| **返青期** | 2月下旬-3月上旬 | 心叶伸长 | 纹枯病、根腐病 |
| **拔节期** | 3月中下旬 | 茎节伸长 | 纹枯病、锈病初期 |
| **孕穗期** | 4月上中旬 | 幼穗形成 | 白粉病、锈病 |
| **抽穗期** | 4月下旬 | 穗子抽出 | 赤霉病、蚜虫 |
| **扬花期** | 5月上旬 | 开花授粉 | **赤霉病高发期** |
| **灌浆期** | 5月中下旬 | 籽粒形成 | 蚜虫、叶枯病 |
| **成熟期** | 6月上旬 | 籽粒成熟 | 穗发芽（如遇雨）|

> 💡 **提示**：不同地区的生育期时间可能相差 ±10-15 天，请根据当地实际情况调整。
