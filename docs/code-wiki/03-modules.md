# 模块文档

## 模块概览

项目包含多个核心模块，每个模块负责特定的功能。以下是主要模块的概览：

| 模块 | 职责 | 位置 |
|------|------|------|
| 视觉模块 | 图像处理和病害检测 | src/vision/ |
| 融合模块 | 多模态信息融合 | src/fusion/ |
| 诊断模块 | 病害诊断逻辑 | src/diagnosis/ |
| 认知模块 | 文本理解和推理 | src/cognition/ |
| 知识图谱 | 知识管理和推理 | src/graph/ |
| Web 模块 | 用户界面和 API | src/web/ |
| 数据模块 | 数据处理和管理 | src/data/ |
| 输入模块 | 输入数据处理 | src/input/ |
| 记忆模块 | 系统记忆和学习 | src/memory/ |
| 部署模块 | 系统部署和优化 | src/deploy/ |

## 模块详细文档

### 1. 视觉模块

**职责**：处理图像输入，使用 YOLO 模型进行小麦病害的目标检测和识别。

**主要功能**：
- 图像预处理：增强、缩放和标准化
- 目标检测：使用 YOLOv8 模型检测病害
- 后处理：结果过滤和优化
- 可视化：检测结果的可视化展示

**关键实现**：
- `vision_engine.py`：视觉引擎的核心实现
- `enhanced_yolo.py`：增强版 YOLO 模型
- `preprocessing.py`：图像预处理功能
- `postprocessing.py`：检测结果后处理

### 2. 融合模块

**职责**：整合多模态信息，包括图像和文本数据的融合。

**主要功能**：
- 跨模态注意力：整合不同模态的信息
- 特征提取：从不同模态中提取特征
- 融合策略：多种融合方法的实现
- 知识图谱增强：利用知识图谱增强融合效果

**关键实现**：
- `fusion_engine.py`：融合引擎的核心实现
- `cross_attention.py`：跨模态注意力机制
- `kad_former.py`：KAD-Former 融合模型
- `kga_module.py`：知识图谱增强模块

### 3. 诊断模块

**职责**：基于融合后的信息进行病害诊断。

**主要功能**：
- 诊断逻辑：基于规则和模型的诊断
- 结果生成：生成诊断结果和置信度
- 报告生成：生成详细的诊断报告
- 治疗建议：基于诊断结果提供治疗建议

**关键实现**：
- `diagnosis_engine.py`：诊断引擎的核心实现
- `report_generator.py`：诊断报告生成器

### 4. 认知模块

**职责**：负责系统的认知和推理能力，使用大语言模型进行文本理解和生成。

**主要功能**：
- 文本理解：理解用户输入的文本描述
- 推理生成：基于上下文生成推理结果
- 提示工程：优化模型提示以提高性能
- 模型管理：管理大语言模型的加载和使用

**关键实现**：
- `cognition_engine.py`：认知引擎的核心实现
- `qwen_engine.py`：Qwen 大语言模型接口
- `prompt_templates.py`：提示模板管理

### 5. 知识图谱模块

**职责**：管理知识图谱，提供知识推理和查询功能。

**主要功能**：
- 知识存储：在 Neo4j 中存储知识图谱数据
- 知识推理：基于图谱进行推理和查询
- 知识更新：支持知识的动态更新和扩展
- 知识检索：高效检索相关知识

**关键实现**：
- `graph_engine.py`：图谱引擎的核心实现
- `knowledge_graph_builder.py`：知识图谱构建器
- `transe_embedding.py`：知识嵌入模型
- `graphrag_engine.py`：GraphRAG 引擎

### 6. Web 模块

**职责**：提供用户界面和 API 接口。

**主要功能**：
- 前端界面：Vue.js 实现的用户界面
- 后端 API：FastAPI 实现的 RESTful API
- 数据管理：用户数据和诊断记录的管理
- 权限控制：用户认证和授权

**关键实现**：
- `web/backend/app/main.py`：后端 API 入口
- `web/frontend/src/App.vue`：前端应用入口
- `web/backend/app/api/`：API 接口实现

### 7. 数据模块

**职责**：负责数据的管理和处理。

**主要功能**：
- 数据加载：加载和解析数据
- 数据预处理：数据清洗和转换
- 数据增强：生成增强数据
- 数据结构：定义和管理数据结构

**关键实现**：
- `data_layers.py`：数据层的核心实现
- `augmentation_engine.py`：数据增强引擎
- `dataset_builder.py`：数据集构建器

### 8. 输入模块

**职责**：统一处理各种输入数据。

**主要功能**：
- 输入验证：验证输入数据的有效性
- 格式转换：转换输入数据为系统内部格式
- 环境编码：编码环境信息
- 输入解析：解析复杂的输入数据

**关键实现**：
- `input_parser.py`：输入解析器
- `input_validator.py`：输入验证器
- `environment_encoder.py`：环境编码器

### 9. 记忆模块

**职责**：存储系统的记忆和历史数据。

**主要功能**：
- 病例记忆：存储和管理病例记录
- 反馈处理：处理用户反馈
- 学习机制：基于历史数据进行学习
- 记忆检索：检索相关的历史数据

**关键实现**：
- `case_memory.py`：病例记忆管理
- `feedback_handler.py`：反馈处理器

### 10. 部署模块

**职责**：负责系统的部署和优化。

**主要功能**：
- 模型量化：模型的量化和优化
- 边缘部署：边缘设备的部署支持
- 导出功能：模型导出和打包
- 性能优化：系统性能的优化

**关键实现**：
- `quantization.py`：模型量化
- `edge_optimizer.py`：边缘部署优化
- `export.py`：模型导出

## 模块间交互

各模块之间通过明确的接口进行交互，形成完整的诊断流程：

1. **输入处理**：输入模块接收用户输入，进行验证和转换
2. **感知处理**：视觉模块处理图像，文本模块处理文本
3. **信息融合**：融合模块整合多模态信息
4. **知识增强**：知识图谱模块提供知识支持
5. **诊断推理**：诊断模块基于融合信息进行诊断
6. **结果输出**：Web 模块展示诊断结果
7. **记忆存储**：记忆模块存储诊断记录

## 模块扩展

每个模块都设计了扩展点，支持功能的扩展和定制：

- **视觉模块**：支持新的视觉模型和检测算法
- **融合模块**：支持新的融合策略和方法
- **诊断模块**：支持新的诊断规则和模型
- **认知模块**：支持新的语言模型和推理方法
- **知识图谱**：支持知识的动态扩展和更新
- **Web 模块**：支持新的前端功能和 API 接口
