# Code Wiki

## 项目概述

本项目是一个小麦病害诊断系统，采用多模态融合技术，结合视觉识别、知识图谱和大语言模型，实现对小麦病害的准确诊断和治疗建议。

## 项目特点

- **多模态融合**：整合图像和文本信息，提高诊断准确性
- **知识驱动**：利用知识图谱增强诊断能力
- **模块化设计**：各模块职责清晰，接口明确
- **可扩展性**：支持模型和功能的扩展
- **实时性**：优化推理速度，支持实时诊断
- **可靠性**：完善的错误处理和容错机制

## 技术栈

- **后端**：Python，FastAPI
- **前端**：Vue.js，TypeScript
- **数据库**：Neo4j（知识图谱），MySQL，Redis
- **机器学习**：PyTorch，YOLOv8，Transformers
- **部署**：Docker，Kubernetes

## 系统架构

项目采用分层架构设计，从底层到上层依次为：

1. **数据层**：负责数据的管理和存储
2. **感知层**：处理输入数据（图像、文本等）
3. **融合层**：整合多模态信息
4. **认知层**：进行推理和决策
5. **应用层**：提供 API 和 Web 界面

## 核心功能

1. **病害检测**：使用 YOLOv8 模型检测小麦病害
2. **多模态融合**：整合图像和文本信息
3. **知识推理**：基于知识图谱进行推理
4. **诊断报告**：生成详细的诊断报告和治疗建议
5. **Web 界面**：提供用户友好的交互界面
6. **API 接口**：支持外部系统集成

## 部署方式

- **本地开发**：直接运行前端和后端服务
- **Docker 部署**：使用 Docker Compose 管理多容器部署
- **边缘部署**：优化模型后部署到边缘设备

## 监控与维护

- **日志管理**：系统日志记录和分析
- **健康检查**：API 健康状态检查
- **性能监控**：使用 Prometheus 和 Grafana 监控系统性能

## 开发工作流

1. **代码修改**：后端修改后自动重载，前端修改后页面自动刷新
2. **测试**：后端测试、前端测试和集成测试
3. **部署流程**：开发测试 → 构建镜像 → 测试环境部署 → 生产环境部署

## 常见问题

- **依赖安装失败**：确保 Python 版本正确，使用虚拟环境隔离依赖
- **服务启动失败**：检查数据库连接和环境变量配置
- **模型加载失败**：确保模型文件存在且路径正确
- **前端无法连接后端**：检查后端服务状态和 API 地址配置
- **性能问题**：启用缓存，优化模型推理速度

## 性能优化

- **模型优化**：使用模型量化，减少模型大小和推理时间
- **服务优化**：使用异步处理，启用缓存，调整工作进程数量
- **数据库优化**：添加索引，使用连接池，定期清理数据
- **部署优化**：使用容器化部署，配置资源限制，使用负载均衡
