# Qwen3-VL-4B-Instruct 全面系统测试 Spec

## Why
当前系统已完成基础功能开发和 Mock 模式测试，但基于 Qwen3-VL-4B-Instruct 多模态大模型的真实诊断功能尚未进行全面系统测试。需要验证模型加载、多模态诊断、边界条件、错误处理、性能表现及兼容性，确保系统符合设计规范和用户需求。

## What Changes
- **验证 Qwen3-VL-4B-Instruct 模型加载**: 确认模型正确加载并可用
- **执行全面功能测试**: 覆盖所有 API 端点和业务流程
- **边界条件测试**: 输入验证、并发、数据量边界
- **错误处理测试**: 异常场景、降级机制、错误响应
- **性能测试**: 响应时间、吞吐量、资源占用
- **兼容性测试**: 浏览器、设备、操作系统兼容性
- **回归测试**: 验证已修复问题未复发
- **生成完整测试报告**: 包含测试结果、缺陷统计、改进建议

## Impact
- 受影响系统：整个 WheatAgent Web 应用系统
- 受影响文件：测试脚本、测试报告文档
- 受影响服务：后端 API、数据库、AI 服务

## ADDED Requirements

### Requirement: Qwen3-VL-4B-Instruct 模型验证
系统 SHALL 正确加载并验证 Qwen3-VL-4B-Instruct 模型：
- 模型文件完整性检查
- 模型加载成功验证
- 推理功能正常验证
- 多模态输入处理验证

#### Scenario: 模型加载成功
- **WHEN** 系统启动并加载 Qwen3-VL-4B-Instruct 模型
- **THEN** 模型应成功加载，显存占用合理（<8GB），推理响应正常

#### Scenario: 多模态诊断
- **WHEN** 用户上传小麦病害图像并输入症状描述
- **THEN** 系统应返回准确的诊断结果，包含病害名称、置信度、防治建议

### Requirement: 功能完整性测试
系统 SHALL 对所有功能模块进行全面测试：
- 用户认证模块（注册、登录、Token 管理、会话管理）
- 诊断服务模块（文本诊断、图像诊断、多模态诊断、历史记录）
- 知识库模块（疾病查询、搜索、分类、详情）
- 统计模块（Dashboard、病害统计、用户统计）
- 系统管理模块（健康检查、监控指标、日志管理）

#### Scenario: 用户完整流程
- **WHEN** 用户完成注册 → 登录 → 诊断 → 查看历史 → 退出
- **THEN** 所有步骤应正常执行，数据一致性保持

### Requirement: 边界条件测试
系统 SHALL 正确处理各种边界条件：
- 输入边界：空值、最大长度、特殊字符、非法格式
- 并发边界：单用户、10并发、50并发、100并发
- 数据量边界：空数据库、100条记录、10000条记录
- 时间边界：Token 过期、会话超时、请求超时

#### Scenario: 输入边界测试
- **WHEN** 用户输入超长文本（>10000字符）或特殊字符
- **THEN** 系统应返回友好错误提示，不崩溃

#### Scenario: 并发测试
- **WHEN** 50个用户同时发起诊断请求
- **THEN** 系统应正常响应，成功率 > 98%

### Requirement: 错误处理测试
系统 SHALL 正确处理各种错误场景：
- 无效输入：错误格式、非法字符、缺失字段
- 认证错误：无效Token、过期Token、权限不足
- 服务错误：数据库连接失败、AI服务不可用、网络超时
- 降级处理：模型未加载时的降级方案

#### Scenario: 认证错误处理
- **WHEN** 用户使用过期 Token 访问受保护资源
- **THEN** 系统应返回 401 错误并提示重新登录

#### Scenario: AI 服务降级
- **WHEN** AI 模型未加载或推理失败
- **THEN** 系统应降级到 Mock 模式或返回友好错误提示

### Requirement: 性能测试
系统 SHALL 满足性能指标要求：
- API 响应时间：健康检查 < 50ms，简单查询 < 100ms，复杂查询 < 500ms
- 诊断响应时间：文本诊断 < 3s，多模态诊断 < 10s
- 并发性能：50并发成功率 > 98%，100并发成功率 > 95%
- 资源占用：CPU < 80%，内存 < 8GB，显存 < 8GB

#### Scenario: API 性能达标
- **WHEN** 执行 API 性能测试
- **THEN** 95% 的请求响应时间应在目标范围内

### Requirement: 兼容性测试
系统 SHALL 兼容主流环境：
- 浏览器：Chrome 120+、Edge 120+、Firefox 120+、Safari 17+
- 设备：桌面端、笔记本、平板、手机
- 操作系统：Windows 10/11、macOS、Linux

#### Scenario: 浏览器兼容
- **WHEN** 在不同浏览器中访问系统
- **THEN** 页面应正常渲染，功能正常使用

### Requirement: 回归测试
系统 SHALL 验证已修复问题未复发：
- P001: Mock 模块导入错误
- P002: bcrypt 密码哈希兼容性
- P003: 缓存参数错误
- P004: Qwen3-VL 模型加载方式
- P005: API 响应格式标准化
- P006: 端口分配规范

#### Scenario: 已修复问题验证
- **WHEN** 执行回归测试
- **THEN** 所有已修复问题应未复发

### Requirement: 测试报告生成
系统 SHALL 生成完整的测试报告：
- 测试概述：目的、范围、环境、方法
- 测试结果：用例统计、通过率、性能数据
- 问题记录：缺陷清单、严重程度、优先级
- 改进建议：优化方向、技术债务、风险评估
- 验收结论：是否达到交付标准

#### Scenario: 测试报告完整
- **WHEN** 所有测试完成
- **THEN** 生成包含所有必要信息的完整测试报告

## 测试目标

1. **功能完整性**: 所有 API 端点功能正常，测试通过率 > 95%

3. **稳定性**: 并发测试成功率 > 98%

6. **文档完整**: 测试报告详细且可操作
